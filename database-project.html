Database Project

This project explores how raw CSV data can be organized into a relational database and queried to better understand customer activity, events, and ticket sales.

Project Overview

The dataset provided for this project consists of three CSV files that represent a small ticketing system. Each file contains information about a different part of the system.

  • Customers – individuals purchasing tickets
  • Shows – events available for purchase
  • Orders – transactions connecting customers to shows

Rather than working with these files as separate spreadsheets, the data was combined into a relational database so that relationships between customers, shows, and orders could be clearly defined and queried.

Database Design

The database was designed using three tables: Customers, Shows, and Orders. Each table has a primary key, and the Orders table uses foreign keys to reference both customers and shows.

This structure creates two one-to-many relationships:

  • One customer can place many orders
  • One show can appear in many orders

This design reduces data duplication and allows information to be retrieved efficiently using joins.

Analytical Questions

To demonstrate how the database can be queried, the following questions were explored:

  • Which shows generated the most revenue?
  • Which customers spent the most on tickets?

SQL Queries

Revenue by Show


SELECT 
  s.title,
  SUM(o.qty * o.price) AS total_revenue
FROM Orders o
JOIN Shows s ON o.show_id = s.show_id
GROUP BY s.title
ORDER BY total_revenue DESC;

Top Customers by Spending


SELECT 
  c.first_name || ' ' || c.last_name AS customer_name,
  SUM(o.qty * o.price) AS total_spent
FROM Orders o
JOIN Customers c ON o.customer_id = c.customer_id
GROUP BY c.customer_id
ORDER BY total_spent DESC;

Results Summary

The results show clear differences in both show popularity and customer spending. Some shows generated significantly more revenue than others, and a small number of customers accounted for a large portion of total ticket sales.

These queries demonstrate how a relational database can be used to extract meaningful insights from structured data.

  2. Identified entities and relationships within the data.
  3. Designed a relational schema with primary and foreign keys.
  4. Created the database using SQLite.
  5. Imported the CSV data into the appropriate tables.
  6. Validated relationships and record counts.
  7. Executed SQL queries to analyze ticket sales and revenue.
